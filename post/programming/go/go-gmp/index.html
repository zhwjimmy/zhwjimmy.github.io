Go系列文章之调度机制

Posted by 松溪 on Sunday, February 12, 2023

Go调度器的由来

进程

  • 进程是资源分配的最小单位。一个进程可以独立运行，并且拥有独立的内存空间
  • 创建进程的开销很大，需要分配独立内存空间。虚拟内存占用几GB。
  • 进程间的数据交互涉及到进程间通信。

线程

  • 线程是程序执行的最小单位。它是进程内部的一个执行流，共享进程的资源，但拥有自己的寄存器和栈
  • 线程相对进程开销小，线程占用几MB
  • 统一进程下的线程共享虚拟内存、全局变量等数据，交换数据比进程方便
  • 上下文切换相对开销更小

并发和并行

协程

  • 内存在操作系统中分为两部分：内核空间和用户空间。内核空间只有内核程序可以访问；用户空间供应用程序访问。
  • 所以线程分成两部分，内核态线程和用户态线程，用户态线程被称为协程，和内核态线程绑定执行
  • 内核调度线程成本比较高，所以分成两部分：内核线程仍然由内核进行调研；用户态线程创建、销毁不需要系统调用，可以自行维护，切换成本小，被称为协程
  • 协程和线程的绑定关系
    • N:1
      • 一旦协程阻塞，会造成线程阻塞
    • 1:1
    • M:N

GMP模型

  • 操作系统调度器调度内核线程
  • 线程和P绑定
  • 每个P有一个P的本地队列，存储goroutine
  • 全局g队列

Go的启动周期

先解释下M0和G0的概念

  • M0，启动程序后的编号为0的主线程，负责初始化第一个goroutine
  • G0，每次启动一个M，都会创建一个goroutine，即G0，仅用于调度其他G，不执行任何函数
  1. 启动程序后，先创建M0线程
  2. M0线程创建G0
  3. 初始化P，P的本地队列，全局G队列等
  4. M0绑定空闲的P，主goroutine加入P的本地队列，M0获取G执行

go指令的调度流程

  1. 创建一个G
  2. 优先加入局部队列；如果本地队列已满，加入全局队列中
  3. M绑定P，从P的本地队列中获取G，执行
  4. 如果时间片超时，G重新放回到P的本地队列尾部
  5. 发生阻塞，操作系统尝试从休眠的M队列中获取一个M，如果没有则新创建一个M，接管P，继续执行P的本地队列；原M放入休眠M队列或者被销毁；G放回到P的本地队列或者全局队列中，等待下次被调度。

调度器的设计策略

  • 复用线程
    • work stealing 机制
      • 当前活跃的线程绑定的P的本地队列没有G，尝试从其他队列偷取一部分G
    • hand off 机制
      • 当前G阻塞，新创建一个M线程绑定P，M加入休眠队列
  • 利用并行
  • 抢占
    • G使用CPU是抢占式，保证尽可能公平
  • 全局G队列

场景罗列

  • 局部性
    • 正在执行的G如果创建新的G，优先加入绑定的P的本地队列
  • G创建了过多的G
    • 优先加入当前P的本地队列
    • 如果队列已满，打乱当前队列，取一半数量的G和新创建的G，放入全局队列中
  • 唤醒正在休眠的M
    • 新创建G，会尝试从休眠线程队列中唤醒一个M
    • 找到P进行绑定；如果未找到重新加入休眠队列
    • P的本地队列中没有G，此时M的状态是自旋线程
    • 优先从全局队列批量获取G
    • work stealing 机制，尝试从其他队列获取一部分G
  • 自旋线程的最大限制
    • 自旋线程+执行线程<=GOMAXPROCS
  • G发生系统调用/阻塞
  • G发生系统调用/非阻塞
    • 尝试绑定原来的P，如果已被占用，尝试获取一个空闲的P
    • 如果未获取到空闲的P，G加入全局队列，M加入休眠队列
