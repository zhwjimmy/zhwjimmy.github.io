如何利用Github Pages和Hugo搭建个人博客

Posted by 松溪的博客 on Thursday, January 5, 2023

如果希望快速、免费的搭建一个博客网站，那么Github Pages + Hugo是一个很好的方案。Hugo是用Go语言编写的博客工具，用户可以使用Markdown编写博客内容，然后通过Hugo生成静态页面；Github Pages是Github提供的一项静态站点托管服务，利用该服务用户可以直接发布网站，无需额外申请域名和服务器。

1. 创建Github Pages

  1. 申请github账号并登录。github地址：https://github.com/
  2. 页面右上角，使用 + 下拉菜单选择“New repository”。
  3. 输入 username.github.io 作为存储库名称。 将 username 替换为你的 GitHub 用户名。 例如，如果用户名为 octocat，则存储库名称应为 octocat.github.io。

此时，Github Pages就已经创建完成，下一步安装Hugo工具。

2. 安装Hugo及创建项目

2.1 安装Hugo

参考官方安装文档

推荐几种安装方式

# macOS
$ brew install hugo

# Go二进制方式
$ go install github.com/gohugoio/hugo
2.2 创建项目
$ hugo new site 项目名
2.3 安装主题

可以挑选喜欢的网站主题并设置。官方主题地址

# 以cleanwhite为例
$ cd 项目
$ git submodule add https://github.com/zhaohuabing/hugo-theme-cleanwhite.git themes/hugo-theme-cleanwhite
$ cp -r themes/hugo-theme-cleanwhite/exampleSite/config.toml config.toml

主题推荐：

  • https://github.com/olOwOlo/hugo-theme-even
  • https://github.com/zhaohuabing/hugo-theme-cleanwhite

tips：主题可以等后面熟悉了hugo项目后再深入研究

2.4 Hugo常用命令
# 新建文章
$ hugo new post/xxx.md

# 本地预览（包含草稿）
$ hugo server -D

# 生成静态文件。生成的静态文件位于public目录下
$ hugo

3. 网站发布到Github Pages

# 将文章生成静态文件
$ hugo // 静态文件位于public目录下

# 进入public目录
$ cd public

# 初始化项目
$ git init
$ git add .
$ git commit -m "init"
$ git remote add origin git@github.com:xxx/xxx.github.io.git 
$ git push -u origin master

# 打开github上xxx项目，此时已经可以看到仓库中已经有代码，代码内容即public中内容
# 访问 xxx.github.io 网址，即可看到对应内容

tips：hugo创建的xxx是源项目，xxx下的public目录是对应的github.io内容。如果xxx也想建立Git仓库的话，推荐使用git submodule方式进行管理。

# 项目根目录下操作
$ rm -rf public
$ git submodule add git@github.com:xxx/xxx.github.io.git public

# 后续变更代码，要提交public子目录和xxx根目录两个项目的代码

# 删除子模块
1. rm -rf 子模块目录 删除子模块目录及源码
2. vi .gitmodules 删除项目目录下.gitmodules文件中子模块相关条目
3. vi .git/config 删除配置项中子模块相关条目
4. rm .git/module/* 删除模块下的子模块目录，每个子模块对应一个目录，注意只删除对应的子模块目录即可
