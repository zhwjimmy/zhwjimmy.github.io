MySQL文章系列（四） 索引

Posted by 松溪 on Wednesday, January 11, 2023
Last Modified on Monday, February 6, 2023

索引的分类

按存储结构

  • B+树索引
  • 哈希索引
  • 全文索引

B+树索引

多路平衡查找树，非叶子节点只存储索引，叶子节点存储数据

B+树 vs B树

B树和B+树都属于多路平衡查找树，保证了查询和修改操作的时间复杂度为对数级别；都是用于磁盘或者其他外存设备的数据组织结构。主要区别如下：

  1. B数的每个节点都可能存储数据；而B+树只有叶子节点存储数据，非叶子节点只存储索引。
    1. B树的空间利用率较高，但非叶子节点能保存的指针数量变小，所以要要保存大量数据，只能增加树的高度，导致 IO 操作变多，查询性能变低。
  2. B+树的叶子节点采用双向链表
  3. B树适用于磁盘或其他外存设备上的数据存储，而B+树更适用于索引数据库和文件系统中的数据存储.

B+树 vs 跳表

B+ 树和跳表都是用于大量数据中提升查询效率的数据结构，查询的时间复杂度为对数级。主要区别如下：

  1. B+树是多路平衡查找树，跳表是链表结构。
  2. B+树主要适合外部存储，可以减少磁盘IO数；跳表比较适合内存存储。

MySQL3层存放2000w数据

  • InnoDB中的数据是按页存储，一页大小是16k
  • 假设主键ID是bigint型，长度是8字节，指针占6字节，一共是14个字节。那么一页可以存1170个指针
  • 假设一行数据大小是是1k，那么1页可以存16行数据
  • 1170 * 1170 * 16 ≈ 2kw

索引的比较

聚簇索引 vs 非聚簇索引

聚簇索引和非聚簇索引是按照物理存储结构进行分类。

聚簇索引指，索引和数据记录存储在一起，B+数的叶子节点存储的是行数据，可以极大的提高查询效率；而非聚簇索引指，叶子节点存放的是索引，大部分情况下需要进行回表查询。

主键索引 vs 唯一索引

  • 主键索引是一种约束，唯一索引是一种索引，二者在本质上是不同的。
  • 主键创建后一定包含一个唯一性索引
  • 唯一索引允许控制，主键索引不允许
  • 主键索引可以被其他表引用为外键，唯一索引不可以
  • 唯一索引可以有多个，主键索引只允许有一个

普通索引 vs 唯一索引

普通索引和唯一索引是不同类型的索引，有着不同的特性和用途。

  • 普通索引是一种普通的索引，字段上可以存在重复的值，主要用于优化查询性能；
  • 唯一索引是一种特殊的所有，索引值在表中要求是唯一的，主要用于保证表中某一列的数据不重复。

普通索引的建立方式比较简单，只需要使用CREATE INDEX语句就可以建立，而唯一索引需要在建立索引时加上UNIQUE关键字。

对于查询过程，普通索引查到满足条件的记录后，需要往后在查直到碰到不满足条件的记录；而唯一索引由于记录唯一，查到第一个满足条件的记录就会停止检索。由于InnoDB的数据是按数据页为单位读写的，查询时会将整页读入内存，因此二者的性能差距微乎其微。

对于更新过程，如果要更新的目标页在内存中，二者的操作基本一致；但如果目标页不在内存中，对于唯一索引，需要把数据页读入内存，判断到没有冲突，插入这个值，语句执行结束；而对于普通索引，将更新记录在change buffer，语句执行就结束了。

核心点：

  • 唯一索引不允许有重复的值
  • change buffer

索引的策略

  • 回表
  • 覆盖索引
    • 可以在非主键索引上查到相关数据，不需要回表查询
  • 索引下推
    • MySQL5.6中，对索引中包含的字段先做判断，直接过滤掉不满足条件的记录，减少回表次数
    • select * from tuser where name like ‘张%’ and age=10 and ismale=1
  • 联合索引
  • 最左前缀原则
    • 可以是联合索引的最左 N 个字段，也可以是字符串索引的最左 M 个字符
