普通索引 vs 唯一索引

普通索引和唯一索引是不同类型的索引，有着不同的特性和用途。

  • 普通索引是一种普通的索引，字段上可以存在重复的值，主要用于优化查询性能；
  • 唯一索引是一种特殊的所有，索引值在表中要求是唯一的，主要用于保证表中某一列的数据不重复。

普通索引的建立方式比较简单，只需要使用CREATE INDEX语句就可以建立，而唯一索引需要在建立索引时加上UNIQUE关键字。

对于查询过程，普通索引查到满足条件的记录后，需要往后在查直到碰到不满足条件的记录；而唯一索引由于记录唯一，查到第一个满足条件的记录就会停止检索。由于InnoDB的数据是按数据页为单位读写的，查询时会将整页读入内存，因此二者的性能差距微乎其微。

对于更新过程，如果要更新的目标页在内存中，二者的操作基本一致；但如果目标页不在内存中，对于唯一索引，需要把数据页读入内存，判断到没有冲突，插入这个值，语句执行结束；而对于普通索引，将更新记录在change buffer，语句执行就结束了。
