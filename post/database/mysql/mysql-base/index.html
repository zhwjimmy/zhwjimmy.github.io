MySQL逻辑架构

  • 网络连接层
    • 与MySQL服务器建立连接
  • 服务层（MySQL Server）
    • 连接池
    • 查询缓存
      • 以key-value对的形式缓存在内存中，命中缓存则直接返回
      • MySQL8.0版本已移除查询缓存功能
    • 分析器
      • 词法分析，语法分析（SQL解析生成一个解析树）
    • 优化器
      • 选择索引，决定表的连接顺序，生成执行计划
    • 执行器
      • 判断是否有表的执行权限，操作引擎提供的接口，返回结果
  • 存储引擎层
  • 系统文件层
    • 文件的物理存储层

MyISAM vs InnoDB

MyISAM 和 InnoDB 是MySQL中两种常用的存储引擎。主要区别如下：

  1. 数据存储结构的区别。
  • MyISAM
    • 数据文件（.MYD）: 存储数据表中所有记录的数据，采用二进制形式存储。
    • 索引文件（.MYI）: 存储数据表中所有索引的数据，采用B+树或Hash索引组织数据。
    • 表信息文件（.frm）: 存储数据表的结构信息，包括表的列信息、类型、长度、约束等。
  • InnoDB
    • 数据文件（.ibd）: 存储数据表中所有记录的数据，采用二进制形式存储。
    • 索引文件（.ibd）: 存储数据表中所有索引的数据，采用B+树组织数据。
    • 表信息文件（.frm）: 存储数据表的结构信息，包括表的列信息、类型、长度、约束等。
  1. 索引的区别。InnoDB支持聚簇索引，MyISAM不支持；InnoDB在索引上只支持B+树，而MyISAM支持B+树和Hash索引；
  2. 锁的区别。InnoDB支持行锁，MyISAM只支持表锁；
  3. 事务的区别。InnoDB支持事务，支持多版本并发控制(MVCC)，MyISAM不支持；
  4. InnoDB支持外键，MyISAM不支持。

trancate、delete和drop的区别

  • 在速度上，一般来说，drop > truncate > delete
  • truncate 和 delete 只删除数据不删除表的结构；drop语句将数据和表结构全部删除，依赖于该表的存储过程、函数将保留，但变为invalid状态
  • truncate自增字段从头开始计数了，而delete的仍保留原来的数值
